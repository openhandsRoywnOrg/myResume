
# AI/ML 基础概念

## 1. 什么是机器学习？

机器学习（Machine Learning, ML）是人工智能的一个子集，它使计算机系统能够通过数据学习和改进，而无需进行明确的编程。

### 核心特点：
- **数据驱动**：通过大量数据训练模型
- **自动学习**：从经验中自动改进性能
- **预测能力**：对新数据做出预测或决策

### 主要类型：
1. **监督学习**：使用标记数据训练
2. **无监督学习**：从无标记数据中发现模式
3. **强化学习**：通过奖励机制学习

## 2. 深度学习与机器学习的区别

| 特性 | 机器学习 | 深度学习 |
|------|---------|---------|
| 数据需求 | 较少 | 大量 |
| 特征工程 | 手动 | 自动 |
| 硬件要求 | 普通 CPU | GPU/TPU |
| 可解释性 | 较高 | 较低 |

## 3. 常见算法

### 监督学习算法：
- 线性回归
- 逻辑回归
- 决策树
- 随机森林
- 支持向量机（SVM）
- 神经网络

### 无监督学习算法：
- K-means 聚类
- 层次聚类
- 主成分分析（PCA）
- 自编码器

## 4. 模型评估指标

### 分类问题：
- **准确率（Accuracy）**：正确预测的比例
- **精确率（Precision）**：预测为正的样本中实际为正的比例
- **召回率（Recall）**：实际为正的样本中被正确预测的比例
- **F1 分数**：精确率和召回率的调和平均

### 回归问题：
- **均方误差（MSE）**
- **平均绝对误差（MAE）**
- **R² 分数**

## 5. 过拟合与欠拟合

### 过拟合（Overfitting）
模型在训练集上表现很好，但在测试集上表现差。

**解决方案：**
- 增加训练数据
- 正则化（L1/L2）
- Dropout
- 早停法（Early Stopping）
- 简化模型

### 欠拟合（Underfitting）
模型在训练集和测试集上都表现差。

**解决方案：**
- 增加模型复杂度
- 增加特征
- 减少正则化
- 训练更长时间

## 常见面试题

### Q1: 如何选择合适的机器学习算法？

**答案：**
1. 首先明确问题类型（分类、回归、聚类等）
2. 考虑数据量和数据质量
3. 考虑特征数量和类型
4. 考虑模型可解释性需求
5. 考虑计算资源和时间限制
6. 通过交叉验证比较多个算法

### Q2: 什么是交叉验证？为什么使用它？

**答案：**
交叉验证是一种评估模型性能的技术，最常用的是 K 折交叉验证。

**优点：**
- 更可靠地评估模型性能
- 充分利用有限的数据
- 减少过拟合风险
- 帮助选择最佳超参数

### Q3: 如何处理不平衡数据集？

**答案：**
1. **数据层面：**
   - 过采样少数类（如 SMOTE）
   - 欠采样多数类

2. **算法层面：**
   - 使用对不平衡不敏感的算法
   - 调整类别权重

3. **评估指标：**
   - 使用 F1 分数、AUC-ROC 等
   - 避免仅使用准确率

### Q4: 解释梯度下降算法

**答案：**
梯度下降是一种优化算法，用于最小化损失函数。

**工作原理：**
1. 初始化参数
2. 计算损失函数的梯度
3. 沿梯度反方向更新参数
4. 重复直到收敛

**变体：**
- 批量梯度下降（BGD）
- 随机梯度下降（SGD）
- 小批量梯度下降（Mini-batch GD）
- Adam、RMSprop 等自适应算法
