## 📋 Issue 解决规范

### AI Agent 工作流程

当使用 `@openhands-agent` 解决 Issue 时：

1. **阅读 AGENTS.md** - 了解项目规范
2. **理解 Issue 需求** - 确认验收标准
3. **查看现有代码** - 保持一致风格
4. **编写测试** - 确保质量
5. **实现功能** - 遵循架构设计
6. **运行测试** - 验证功能
7. **提交代码** - 清晰的 commit 信息

### Issue 模板使用

- **功能请求** → 使用 `feature_request.md`
- **Bug 报告** → 使用 `bug_report.md`
- **AI 任务** → 使用 `ai_task.md`

## 💡 关键建议

### 1. 渐进式重构
不要一次性重写所有代码，而是：
- 小步迭代
- 每个功能模块单独重构
- 保持向后兼容
- 充分测试

### 2. 测试先行
- 先写测试，再写代码
- 保持高测试覆盖率
- 自动化回归测试
- CI/CD 自动验证

### 3. 文档驱动
- 先写文档，再实现功能
- 保持文档更新
- 代码即文档（清晰的命名和结构）
- 示例代码

### 4. AI 协作
- 使用 AI 生成初始代码
- 人工 Review 和优化
- 遵循项目规范
- 保持代码一致性
