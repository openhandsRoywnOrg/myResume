## 📝 文案指南

### 语调风格
- **专业** (Professional) - 但不生硬
- **友好** (Friendly) - 但不随意
- **激励** (Inspiring) - 但不浮夸
- **清晰** (Clear) - 直接表达价值

### 用词建议
- ✅ 使用主动语态
- ✅ 强调价值和结果
- ✅ 避免技术 jargon 堆砌
- ✅ 保持简洁有力

### 示例对比
```
❌ "We provide comprehensive learning materials for DevOps"
✅ "Master DevOps. Land Your Dream Job."

❌ "This website contains many knowledge points about AI"
✅ "Your AI DevOps Knowledge Base"
```

## 📊 Code Wiki 布局分析总结

基于对 https://codewiki.google/github.com/modelcontextprotocol/python-sdk 的深入分析，总结以下可借鉴的布局优势：

### 1. 三栏布局结构 ⭐⭐⭐⭐⭐

```
┌──────────┬──────────────────┬──────────┐
│ 左侧导航 │   主内容区域      │ 右侧目录 │
│ 280px    │   自适应          │ 240px    │
│ 树状结构 │   文章內容        │ On this  │
│          │                  │ page     │
└──────────┴──────────────────┴──────────┘
```

**优势**:
- 左侧树状导航：清晰展示知识点层级关系
- 右侧目录导航：快速跳转到当前页面的任意章节
- 主内容区域：专注于阅读，无干扰

**实施优先级**: 高  
**详细设计**: 查看 [DOC_DRIVEN_DESIGN.md](docs/DOC_DRIVEN_DESIGN.md)

---

### 2. 章节锚点链接 ⭐⭐⭐⭐

**特性**: 每个标题旁显示 `link` 图标，悬停时可见

**优势**:
- 支持直接分享特定章节链接
- 增强内容的可导航性
- 提升专业感

**实施示例**:
```html
<h2>
  <a href="#section-id" class="anchor-link">
    <span class="icon">link</span>
  </a>
  章节标题
</h2>
```

**实施优先级**: 中

---

### 3. 代码源码跳转 ⭐⭐⭐⭐⭐

**特性**: 代码中的类名、函数名可点击跳转到 GitHub 源码

**优势**:
- 理论与实践结合
- 方便查看完整实现
- 增强学习体验

**实施示例**:
```html
<span class="class-name" data-link="/topics/base-model">Topic</span>
```

**实施优先级**: 高（对知识点学习极有价值）

---

### 4. 交互式图表 ⭐⭐⭐⭐

**特性**: 支持缩放、拖拽的架构图/流程图

**优势**:
- 复杂概念可视化
- 支持细节查看
- 提升专业度

**实施示例**:
- 文档驱动开发流程图
- 系统架构图
- 技术栈关系图

**实施优先级**: 中

---

### 5. 更新时间显示 ⭐⭐⭐

**特性**: 在文章头部显示最后更新时间和 commit 版本

**优势**:
- 增加内容可信度
- 便于追踪内容变化
- 体现项目活跃度

**实施示例**:
```html
<div class="meta-info">
  <span>Last updated: 2025-03-03</span>
  <span>Based on commit <a href="#">abc123</a></span>
</div>
```

**实施优先级**: 低

---

### 6. AI 辅助入口 ⭐⭐⭐⭐⭐

**特性**: 右下角固定 AI 聊天入口

**优势**:
- 即时解答疑问
- 增强互动性
- 符合 AI 时代特征

**实施示例**:
```
🤖 悬浮按钮 → 点击展开聊天面板
```

**实施优先级**: 高（与项目 AI 定位契合）

---

### 7. 响应式导航 ⭐⭐⭐⭐

**特性**: 根据屏幕尺寸自动调整布局

**断点**:
- Desktop (≥1200px): 三栏布局
- Tablet (768-1199px): 双栏布局（隐藏右侧目录）
- Mobile (<768px): 单栏布局（隐藏两侧导航，使用汉堡菜单）

**实施优先级**: 高
